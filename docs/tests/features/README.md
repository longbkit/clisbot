# Feature Test Folders

## Purpose

Use `docs/tests/features/` to keep test specs aligned with the same stable feature names used in `docs/features/` and `docs/tasks/features/`.

## Placement Rules

Put a test spec here when:

- the behavior belongs to one feature area
- humans should be able to run it directly
- future automation should be able to derive a test from it

Keep the structure shallow and use one folder per stable feature area.
