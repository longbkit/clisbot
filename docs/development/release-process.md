# clisbot Release Process

The canonical release workflow lives in [`skills/release-clisbot/SKILL.md`](../../skills/release-clisbot/SKILL.md).

Use that skill for beta releases, stable/latest releases, npm auth, release notes, migration notes, GitHub Releases, tags, validation, and post-publish checks. Do not duplicate the release command sequence here; keeping one owner avoids stale publish instructions.

## Operator Notes

- For AI-assisted releases, ask the agent to use `release-clisbot`.
- For manual review, read [`skills/release-clisbot/SKILL.md`](../../skills/release-clisbot/SKILL.md) directly before publishing.
- Keep npm publish discipline aligned with `AGENTS.md`.
- Update this page only when the release workflow owner changes.
