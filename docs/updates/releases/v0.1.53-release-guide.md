# clisbot v0.1.53 Release Guide

Use this guide when updating from `v0.1.52` or an earlier stable install.

## What Changed

`v0.1.53` is the stable release for the channel/control/config/agents cleanup that was tested through the `0.1.53-beta.*` line.

The main things to notice:

- queue, loop, and `message-tool` settlement behavior is more consistent
- Slack, Telegram, and Zalo Bot keep recent context cleaner around slash/control commands
- Zalo Bot setup docs now include the Bot Creator QR code and official create-bot link
- Zalo Personal docs now cover safe defaults, media attachments, media groups, and operator channel actions more clearly
- the root README and user guides now cover more channels and localized docs
- existing admin roles receive the new sensitive channel permissions during the automatic config update

## Update

```bash
npm install -g clisbot@0.1.53
clisbot restart
clisbot status
```

Or use the stable dist-tag:

```bash
npm install -g clisbot@latest
clisbot restart
clisbot status
```

## Migration

Manual action: none.

Configs before schema `0.1.53` are backed up and rewritten automatically to add:

- `contactsManage`
- `groupsManage`
- `sensitiveChannelActionManage`

These permissions are for admin-only sensitive channel-native actions such as contact management, group management, poll mutations, or voter-revealing poll reads.

## What To Check After Restart

```bash
clisbot --version
clisbot status
```

Expected version: `0.1.53`.

Check the channels you actively use:

- Telegram and Slack should show `connection=active` when enabled.
- Zalo Bot and Zalo Personal should show active only when configured and enabled.
- `Active runs` should be empty unless work is currently running.
