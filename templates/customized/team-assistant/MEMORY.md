# MEMORY.md - Team Long-Term Memory

Use this file for shared long-term context in a team assistant workspace.

Good content:

- project background worth reusing
- architecture or product decisions
- team norms
- handoff context
- lessons learned
- recurring operational facts
- common failure modes

Avoid:

- private personal details that do not belong in a shared workspace
- credentials or sensitive secrets unless you explicitly intend to store them here
- one-off noise better kept in daily notes
